[![CI](https://github.com/riccardodicurti/salient-custom-elements/actions/workflows/ci.yml/badge.svg)](https://github.com/riccardodicurti/salient-custom-elements/actions/workflows/ci.yml)

Dev/staging generator for **native WPBakery elements** integrated with the commercial **[Salient](https://themeforest.net/item/salient-responsive-multipurpose-theme/4363266)** theme. Create elements with AI or manually, preview instantly, review generated PHP, and ship a production-only mini-plugin.

> **Italian translation included** — source language is English; when WordPress is set to `it_IT`, the bundled `languages/salient-custom-elements-it_IT.mo` loads automatically.

## Features

- **AI generation** via the [WordPress AI plugin](https://wordpress.org/plugins/ai/) with streaming progress
- **Multi-turn AI editing** — chat on the edit screen to refine existing elements
- **Original prompt preserved** — the generation prompt is saved and shown when editing
- **Instant WPBakery integration** — generated PHP is loaded immediately; elements appear in the editor
- **Live preview** — private preview page per element
- **Production rules** — SEO, security, accessibility (WCAG 2.2 AA), responsive CSS enforced in code and AI context
- **Built-in auditor** — flags heading, alt, noopener, inline handlers, and more during review
- **Ship to production** — package **all elements** into standalone `salient-shipped-elements` and remove this generator
- **Internal wiki** — rules, Salient option bindings, WPBakery param types, and runtime `nectar_*` reference

## Requirements

| Component | Version |
|-----------|---------|
| WordPress | 6.9+ (7+ recommended for AI) |
| PHP | 8.0+ |
| Salient theme | Active |
| WPBakery Page Builder | Active |
| WordPress AI plugin | Optional (required for AI generation) |

This plugin does **not** include or redistribute Salient or WPBakery. Dependency checks block activation when they are missing and show an admin notice at runtime if they disappear later.

## Workflow

1. **Create** — describe an element in natural language (AI) or fill the form manually.
2. **Use immediately** — the plugin writes a standalone PHP file and loads it; the element is available in WPBakery.
3. **Preview & review** — open the edit screen, use the preview button, read generated PHP and auditor warnings.
4. **Refine with AI** — use the multi-turn chat below the form to apply changes; fields update automatically.
5. **Export** — download a zip or use **Package and remove generator** to install `salient-shipped-elements` and delete this dev tool.

Permissions: users with `edit_pages` can create and edit; `manage_options` can export, ship, and delete.

## Installation

```bash
git clone https://github.com/riccardodicurti/salient-custom-elements.git
cp -r salient-custom-elements /path/to/wp-content/plugins/
```

Activate in **Plugins**. Open **Custom Elements** in the admin menu.

## AI setup

AI generation uses the official WordPress AI plugin. Install and configure a model in **Settings → AI**. The plugin connects via the `sce_ai_provider` filter (wired to `WordPress\AI\get_ai_service()` when available).

Without the AI plugin, create and edit elements manually — all other features work.

## Generated code

Each element becomes a self-contained PHP file that depends only on WordPress, WPBakery, and Salient:

- Escaped output by type (`esc_html`, `esc_attr`, `esc_url`, `wp_kses_post`)
- Semantic HTML5, configurable heading levels (h2–h4)
- Accessible images, focus styles, reduced-motion support
- Scoped responsive CSS generated by the plugin (not duplicated in templates)
- `ABSPATH` guard and prefixed functions

## Translations

- Text domain: `salient-custom-elements`
- Template: `languages/salient-custom-elements.pot`
- Bundled Italian: `languages/salient-custom-elements-it_IT.mo`

Regenerate translations after string changes:

```bash
php tools/build-translations.php
```

## Continuous integration

GitHub Actions runs on every push and pull request to `main`/`master`:

| Job | Tool | Purpose |
|-----|------|---------|
| **PHPCS** | [10up/wpcs-action](https://github.com/10up/wpcs-action) + `phpcs.xml.dist` | WordPress Coding Standards |
| **Plugin Check** | [Plugin Check](https://wordpress.org/plugins/plugin-check/) via wp-env | WordPress.org readiness and security checks |

Commercial dependencies (Salient + WPBakery) are stubbed in `.ci/stubs/` so the plugin can activate in CI. Production sites still require the real theme and builder.

## File structure

```

includes/
  class-sce-dependencies.php    Salient + WPBakery checks
  class-sce-salient.php         Theme options + WPBakery map extraction
  class-sce-element-store.php   Element definitions (CPT)
  class-sce-rules.php           Production rules + template audit
  class-sce-code-generator.php  Definition → standalone PHP
  class-sce-loader.php          Load generated files + export mini-plugin
  class-sce-ai-generator.php    AI prompts and JSON schema
  class-sce-generator.php       Generate/modify orchestration
  class-sce-rest.php            Streaming REST endpoint
  class-sce-preview.php         Private preview pages
  class-sce-admin.php           Admin UI
  class-sce-wiki.php            Rules wiki
  class-sce-plugin.php          Wiring, textdomain, seed example
languages/                      POT, PO, MO
assets/
  admin.css
  admin-generate.js             AI chat (wp-i18n + localized strings)
uninstall.php                   Cleanup on uninstall
```
